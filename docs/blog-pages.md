# Blog & long-form research page formats

This document defines the two content page layouts used for **Learn** content:
blog articles (`resources/` and most `guides/`) and **long-form research**
pages (interactive research with full-bleed sections).

Canonical references:

| Format | Example file |
| ------ | ------------ |
| **Blog** | `guides/google-vehicle-listing-ads.html` |
| **Long-form research** | `guides/product-data-feeds-as-keywords.html` |
| **Blog (generated)** | `resources/how-much-to-budget.html` |
| **Blog lead-magnet** | `guides/conversion-tracking-shopify-2026.html` |

Shared building blocks live in `styles.css`. Header and footer come from
`site_nav.py` (or are inlined in hand-edited guide files). The newsletter block
is `blog_newsletter(rel)` in `site_nav.py`.

## Shared rules (both formats)

### Page order

Every blog and long-form research page ends with **only** these two blocks:

1. **Newsletter CTA** ŌĆö `<section class="newsletter-cta" id="newsletter">`
2. **Site footer** ŌĆö `<footer class="site-footer">`

Do **not** add:

- `section.section--alt` ŌĆ£Contact UsŌĆØ or ŌĆ£Need help?ŌĆØ CTAs
- `section.article-related` ŌĆ£Related ResourcesŌĆØ grids
- `section.article-hero` / `section.article-feature-image` (legacy layout)

### Hero (split layout)

Both formats use the same hero:

```html
<section class="hero">
  <div class="container hero-split">
    <div class="hero-text">
      <span class="eyebrow">Free Guide &middot; {topic}</span>
      <h1>{title}</h1>
      <p class="lead">{one- or two-sentence summary}</p>
      <div class="hero-actions">
        <a href="#article-body" class="btn btn-dark">{primary CTA}</a>
        <!-- optional secondary link -->
      </div>
    </div>
    <div class="hero-video">
      <!-- video OR image ŌĆö see media rules below -->
    </div>
  </div>
</section>
```

**Hero media rules** (right column):

| Condition | Use |
| --------- | --- |
| Page has a YouTube (or other) embed URL | `<div class="video-frame"><iframe ŌĆ”></iframe></div>` inside `.hero-video` |
| No video, but page has a feature/hero image | `<img class="guide-hero-image" ŌĆ”>` inside `.hero-video` |
| Video exists | Do **not** also show the feature image in the hero; video takes priority |
| No video | Do **not** duplicate the image in the article body unless it illustrates a specific point inline |

Optional secondary action in `.hero-actions`:

- `hero-arrow-link` to YouTube (`video_watch_url` in generator data)
- Anchor link to an interactive demo (`#keyword-demo`, etc.)

`../` paths for nested folders):


## Format 1: Blog

**Use for:** Standard guides and resource articles ŌĆö step-by-step tutorials,
explainers, and posts with optional inline tools.

**Locations:**

- `resources/<slug>.html` ŌĆö generated by `generate_pages.py` ŌåÆ `render_article()`
- `guides/*.html` ŌĆö hand-maintained guides (except long-form research)

### Body structure

**Simple blog** (most `resources/` pages):

```html
<section class="article-body" id="article-body">
  <div class="container">
    <!-- prose: h2, h3, p, ul, figure, blockquote -->
    <!-- optional: section.interactive-tool -->
  </div>
</section>
```

**Blog with table of contents** (long step-by-step guides):

```html
<section class="article-body article-body--with-toc" id="article-body">
  <div class="container guide-body-split">
    <aside class="guide-toc-sidebar">
      <nav class="guide-toc" aria-label="Table of contents">ŌĆ”</nav>
    </aside>
    <div class="guide-body-content">
      <!-- optional: .guide-aside-block (text + image side by side) -->
      <!-- prose sections with id anchors matching TOC -->
    </div>
  </div>
</section>
```

Reference: `guides/google-vehicle-listing-ads.html`.

### Optional blog variants

**Lead-magnet blog** ŌĆö hero + gated form instead of full article body:

```html
<section class="section section--alt" id="get-the-guide">
  <div class="container" style="max-width:680px;">
    <!-- HubSpot embedded form -->
  </div>
</section>
```

Reference: `guides/conversion-tracking-shopify-2026.html`. Still uses the split
hero (video in `.hero-video`). Newsletter + footer follow the form section.

### Generator data (`RESOURCES` in `generate_pages.py`)

| Key | Required | Notes |
| --- | -------- | ----- |
| `slug`, `title`, `lead`, `meta_description`, `date` | yes | |
| `feature_image`, `feature_alt` | yes | Hero image when no `video_embed` |
| `categories` | yes | First category name feeds default eyebrow |
| `body_html` | yes | HTML inside `.container` |
| `eyebrow` | no | Overrides default `Free Guide ┬Ę {category}` |
| `video_embed` | no | YouTube embed URL ŌåÆ hero iframe |
| `video_watch_url` | no | Adds ŌĆ£Watch on YouTubeŌĆØ link in hero |
| `hero_cta_href`, `hero_cta_label` | no | Default `#article-body` / `Read the Guide` |

Regenerate after edits:

```bash
python3 generate_pages.py
```

### Scripts

| Page type | Script |
| --------- | ------ |
| Articles with inline images | `scripts/guide-image-lightbox.js` |
| Budget calculator | Inline JS in `body_html` (`resources/how-much-to-budget.html`) |

## Format 2: Long-form research

**Use for:** In-depth research articles with **full-bleed interactive sections**
that break out of the narrow prose column.

**Canonical page:** `guides/product-data-feeds-as-keywords.html` (ŌĆ£Where Product
Data Feeds Are UsedŌĆØ).

### Body structure

```html
<section class="article-body article-body--long-form-research" id="article-body">
  <div class="container">
    <!-- opening prose -->
  </div>

  <section class="feed-demo-showcase" id="keyword-demo" data-tool="feed-keyword-matcher" ŌĆ”>
    <div class="feed-demo-showcase-inner">
      <!-- full-width interactive demo -->
    </div>
  </section>

  <div class="container">
    <!-- closing prose -->
  </div>
</section>
```

Key differences from standard blog:

| Blog | Long-form research |
| ---- | ------------------ |
| Single `.container` for entire body | Multiple `.container` blocks with **full-bleed** sections between them |
| TOC sidebar optional | No TOC sidebar |
| `interactive-tool` inside container | Large showcase blocks (e.g. `.feed-demo-showcase`) as direct children of `.article-body` |
| Class: `article-body` | Class: `article-body article-body--long-form-research` |

The hero follows the same split layout as blog (image in `.hero-video` for this
page; no video).

### Scripts

| Tool | Script |
| ---- | ------ |
| Feed keyword matcher demo | `scripts/feed-keyword-matcher.js` |

See also `docs/feed-demo-design-preferences.md` for showcase styling notes.

## Checklist for new pages

### New resource article (blog)

1. Add dict to `RESOURCES` in `generate_pages.py`.
2. Set `video_embed` if the page has a hero video; otherwise `feature_image` is used.
3. Run `python3 generate_pages.py`.
4. Add card on `resources.html` and URL in `sitemap.xml`.
5. Confirm: split hero ŌåÆ `article-body` ŌåÆ newsletter ŌåÆ footer. No related/contact blocks.

### New long-form research page

1. Copy structure from `guides/product-data-feeds-as-keywords.html`.
2. Use `article-body--long-form-research` on the body section.
3. Place full-bleed showcase sections as siblings of `.container` divs inside `.article-body`.
4. End with newsletter + footer only.
5. Add card on `resources.html` and URL in `sitemap.xml`.

### Migrating legacy articles

Remove these legacy sections if present:

- `section.article-hero`
- `section.article-feature-image`
- `section.article-related`
- `section.section--alt` contact CTAs
- `.article-categories-footer` (optional; no longer generated)

Move feature image into hero right column; move any body-embedded video into hero.
